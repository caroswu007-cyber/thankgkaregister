# 唐卡传承公益体验课 - 报名系统开发计划

## 技术栈

- 纯 HTML + CSS + 原生 JavaScript（无框架，无需服务器）
- EmailJS（免费版 200封/月，足够 108 人报名）
- 本地 localStorage 辅助存储剩余名额状态（管理员端控制）
- `data/students.json`：学员历史数据文件（由管理员从腾讯文档导出更新）

## 页面结构（6个文件）

```
报名系统/
├── index.html           # 课程介绍 + 课程列表 + 温馨提示
├── register.html        # 报名表单页
├── success.html         # 报名成功页（含确认信息）
├── query.html           # 学员自助查询页
├── admin.html           # 管理员后台（密码保护）
└── data/
    └── students.json    # 学员数据文件（管理员维护）
```

## 各页面详细说明

### index.html（课程列表页）

- 顶部导航：课程报名 / 关于我们
- **温馨提示区块**（参考桂东中心风险提示模式，内容来自图片）：
  - 全程参与五天课程
  - 年龄需 16 岁以上
  - 活动免费提供所有绘画工具、矿物颜料
  - 本次活动限 **108** 位名额，先到先得
  - 课程期间手机须放置于"养机场"
  - 全程不杀生、不偷盗、不邪淫、不妄语、不饮酒
  - 全天茹素
  - 参加活动请扫码添加微信，备注"唐卡"
- **课程卡片**：展示课程信息（时间/地点/讲师/剩余名额）
- 报名按钮（名额满时显示"已满员"并禁用）
- 底部：微信二维码 + 联系方式

### register.html（报名表单页）

报名人填写字段：

- 姓名（必填）
- 手机号（必填）
- 微信号（必填）
- 年龄（必填，需 ≥16）
- 电子邮件（必填，用于发送确认邮件）
- 性别
- 所在城市
- 是否有绘画基础（是/否）
- 健康声明勾选框（已征得家属同意 / 身体健康无慢性病 / 知晓活动规则）
- 提交按钮

提交逻辑：

1. 前端验证所有字段
2. 调用 EmailJS → 发送确认邮件给报名者
3. 调用 EmailJS → 发送通知邮件给管理员（含完整信息）
4. 跳转到 success.html

### success.html（成功页）

- 显示报名成功提示
- 复述课程时间、地点
- 提醒添加微信二维码备注"唐卡"
- 提醒查收确认邮件
- 返回首页按钮

### query.html（学员自助查询页）

查询入口：

- 输入**姓名** + **身份证号**（后6位用于匹配，全号不明文存储）
- 点击查询按钮

查询结果展示：

- **本期报名状态**：已确认报名 / 候补 / 未报名
- **历史参课次数**：共参加过 X 期唐卡课
- **历史参课记录**：每期期次 + 日期（列表形式）
- **联系信息确认**：显示登记的手机号（中间4位隐藏）、微信号

数据来源：读取 `data/students.json`（由管理员从腾讯文档导出并更新）

隐私保护：

- 身份证号只用于本地匹配比对，不回显完整号码
- JSON 文件中身份证存储为哈希值（SHA-256 前端运算）

### admin.html（管理员后台）

- 密码登录（简单前端密码，保护页面）
- 显示当前已报名人数 / 剩余名额（从 localStorage 读取）
- 一键开启/关闭报名
- 数据同步说明：如何从腾讯文档导出并更新 `data/students.json`

## EmailJS 邮件模板

**确认邮件（发给报名者）：**

- 主题：「唐卡传承公益体验课」报名确认
- 内容：报名信息摘要 + 课程时间地点 + 温馨提示全文 + 添加微信说明

**通知邮件（发给管理员）：**

- 主题：新报名通知 - 姓名 手机号
- 内容：完整报名信息（可直接复制到腾讯文档）

## 设计风格

- 色调：米黄/暗金/藏红，呼应唐卡藏族艺术风格
- 字体：中文衬线/宋体感，庄重典雅
- 整体氛围：传统文化 + 公益感
- 移动端优先（主要用户在手机上查看）

管理员工作流程：

1. 在腾讯文档中维护学员信息（正常操作）
2. 需要更新网站查询数据时：将腾讯文档导出为表格 → 使用管理员页面提供的**一键转换工具**生成 JSON → 替换 `data/students.json` 文件

## EmailJS 配置说明（需用户操作）

用户需要：

1. 注册 [EmailJS](https://www.emailjs.com) 账号（免费）
2. 添加邮件服务（如 QQ邮箱 / 163邮箱）
3. 创建两个邮件模板（确认模板 + 管理员通知模板）
4. 将 `Service ID` / `Template ID` / `Public Key` 填入代码中的配置区域
